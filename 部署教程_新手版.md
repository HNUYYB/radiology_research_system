## 🔧 二、准备工作（在你自己的电脑上操作）

### 2.1 下载安装 XShell

如果你还没有 XShell：

1. 访问 https://www.xshell.com/zh/free-for-home-school/
2. 下载 "XShell 免费版"
3. 安装（一路下一步即可）

1. 打开 XShell
2. 点击左上角 **"文件" → "新建"**
3. 填写：
   - **名称**：`A6000服务器`（随便起个名字）
   - **主机**：`3.tcp.vip.cpolor.cn`
   - **端口号**：`22`
4. 点击 **"连接"**
5. 弹出登录框：
   - **用户名**：`user01`
   - **密码**：`admin@123`
6. 登录成功后，你会看到类似这样的命令行提示符：
   ```
   user01@a6000:~$
   ```

## 📦 三、上传代码到服务器

### 方法 A：用 Git 克隆（推荐）

在 XShell 的黑色窗口里，复制粘贴下面这一行命令，然后按回车：

```bash
cd ~
git clone https://github.com/HNUYYB/radiology_research_system.git
```

等一会儿就克隆完成了。

### 方法 B：用 pscp 命令上传（如果 Git 不行）

在你自己电脑（Windows）上：

1. 按 `Win + R`，输入 `cmd`，回车
2. 在弹出的黑色窗口里输入：

```cmd
cd E:\radiology_research_system
pscp -r -P 22 -l user01 -pw "admin@123" . 3.tcp.vip.cpolor.cn:/home/user01/radiology_research_system/
```

> ⚠️ 注意：用 `admin@123` 这个密码时，`@123` 前面有个空格是为了安全。直接复制上面的命令即可。

如果 pscp 找不到，先去 PuTTY 官网下载：https://www.putty.org/ （pscp 和 putty 是同一个安装包）

## 🐍 四、创建虚拟环境 yybvenv

> **什么是虚拟环境？**
> 想象你有一个工具箱，专门放这个项目用的工具和零件，不会和其他项目混在一起。`yybvenv` 就是这样一个独立的 Python 工具箱。

在 XShell 里，依次执行以下命令（每行输完按回车）：

```bash
# 进入项目目录
cd ~/radiology_research_system

# 创建虚拟环境，名字叫 yybvenv
python3 -m venv yybvenv
```

如果提示 `python3: command not found`，试试用 `python` 代替：
```bash
python -m venv yybvenv
```

创建完之后，激活虚拟环境（告诉系统"我现在要用这个工具箱了"）：

```bash
source yybvenv/bin/activate
```

激活成功后，你会看到命令提示符前面多了一行 `(yybvenv)`，像这样：
```
(yybvenv) user01@a6000:~/radiology_research_system$
```

> 💡 **重要**：以后每次重新连接服务器，都要重新运行这条命令来激活虚拟环境：
> ```bash
> source ~/radiology_research_system/yybvenv/bin/activate
> ```

## 📥 五、安装依赖包

确保虚拟环境已激活（看到 `(yybvenv)` 前缀），然后运行：

```bash
pip install --upgrade pip
```

等完之后，安装项目需要的所有依赖：

```bash
pip install -r backend/requirements.txt
```

这一步可能需要 2-5 分钟，等它跑完。如果看到 `Successfully installed ...` 就说明装好了。

## 🚀 八、启动服务

有两种启动方式，选择其中一种：

### 方法 1：直接用 Flask（简单，适合测试）

```bash
cd backend
python app.py
```

服务启动后，你会看到类似：
```
 * Running on all addresses (0.0.0.0)
 * Running on http://0.0.0.0:8013
```

### 方法 2：用 Gunicorn（推荐，更稳定）

回到项目根目录：
```bash
cd ~/radiology_research_system
gunicorn --worker-class eventlet -w 1 --bind 0.0.0.0:8013 --timeout 600 --access-logfile ./access.log --error-logfile ./error.log --chdir backend app:app
```

> 这会后台运行服务，关闭 XShell 服务也会继续跑。

## 🔄 十、停止/重启服务

### 停止服务
```bash
pkill -f 'gunicorn.*8013'
```

### 重启服务
```bash
cd ~/radiology_research_system
pkill -f 'gunicorn.*8013' 2>/dev/null; sleep 1
gunicorn --worker-class eventlet -w 1 --bind 0.0.0.0:8013 --timeout 600 --chdir backend app:app &
```

### 查看日志
```bash
tail -f ~/radiology_research_system/error.log
```

## 🛠️ 十一、常见问题排查

### 问题 1：`python3: command not found`
解决：用 `python` 代替 `python3`

### 问题 2：端口被占用
```bash
# 查看哪个进程占用了 8013 端口
lsof -i :8013
# 或者直接用 kill 杀掉
pkill -f 'gunicorn.*8013'
```

### 问题 3：连接数据库失败
```bash
# 检查 backend/instance 目录是否存在
ls -la backend/instance/
# 如果不存在，创建它
mkdir -p backend/instance
```

### 问题 4：服务启动了但访问不了
检查服务器防火墙是否开放了 8013 端口，联系服务器管理员确认。

### 问题 5：想更新代码
```bash
cd ~/radiology_research_system
git pull origin main
pkill -f 'gunicorn.*8013' 2>/dev/null; sleep 1
gunicorn --worker-class eventlet -w 1 --bind 0.0.0.0:8013 --timeout 600 --chdir backend app:app &
```
